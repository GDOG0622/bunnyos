# QQ App — Telegram 化改造 · 需求规格与路线图

> 本文是 2026-06-18 与用户对齐十轮后的结论汇总。作为后续开发的唯一依据。
> 原则：**只换皮 + 加功能，保留所有现有功能**；AI 用用户自己的 OpenAI 兼容接口（全局主 API）。

---

## 2026-06-19 最新状态

- 阶段 0 视觉换皮已完成首版：Telegram 色系、圆头像、横屏左 rail + 右主区域、头栏在线/正在输入、日期分割线、会话列表末条时间、气泡外侧平行时间/已读、表情常驻、其它工具收进 `+`。
- 阶段 1 消息交互骨架已接入：长按/右键气泡上方浮出图标条，点外部关闭；支持重新生成、气泡内编辑、收藏、回复引用、删除。
- 回复引用和编辑都作用于**单条气泡**；一次 AI 生成出的若干短气泡视为**回复组**，重新生成、`>` 生成新版本、`← 1/3 →` 切换都作用于整个回复组。
- 重新生成、`>` 和版本导航只允许出现在**当前聊天最后一个 AI 回复组/最后一条 AI 信息**上；历史 AI 气泡不显示这些按钮。
- 输入栏 `+` 工具区已有垃圾桶删除模式：可逐条选择，也可勾选“从选中楼层往下”后选择该楼层及之后全部消息；删除前确认。
- 表情面板改为按需懒加载 `emoji-picker-element`，点击任意面板外区域关闭，用于降低 QQ/App 打开时的初始卡顿。
- 点开对话后，宽屏左侧聊天列表会自动收起；用户仍可再次点击“消息”tab 展开/收起。
- User 人设系统已接入：「我」页展示、头像入口、个性签名原地编辑、编辑自动保存、切换账号、新建/删除、当前人设注入 AI prompt。
- 提示词管理已并入 QQ「我」页，使用 SillyTavern 兼容预设工作副本；QQ 当前预设保存到 `data/qq/settings.json`，`/api/qq/reply` 会按 `prompt_order` 装配启用条目。
- 当前酒馆预设顶部可编辑采样参数，QQ 回复会读取 `temperature`、`top_p`、`frequency_penalty`、`presence_penalty`、`openai_max_tokens`，设置 App 里不再管理主 API 这些参数。
- 世界书已改为“本”级管理：支持新建/重命名/删除整本、编辑条目、导入 SillyTavern 世界书 JSON、选择 QQ 全局世界书，并在好友资料页绑定角色世界书。
- QQ 回复已能展开世界书 marker：QQ 全局世界书进入 `<world_info>`，角色绑定世界书进入 `<memories>`；预设装配预览可展开这两类实际内容。
- 当前仍未完成：reactions 表情回应、回复引用注入 AI 提示词、世界书关键词/条件命中和 token 控制、NSFW 小节、表情包注入与渲染、角色卡 PNG 导入、开场白 swipe、主动消息等。

## 一、视觉（Telegram 风 · 已锁定）

- 风格：Telegram，**跟随系统明暗**（prefers-color-scheme）。
- 气泡：对方=白（左），我=**Telegram 浅绿 `#EFFDDE`**（右）；暗色有对应深色。收尾角圆角。
- 气泡元素：**时间戳**（用消息 created_at）+ 我的消息带 **已读双勾 ✓✓**（装饰）；时间/已读放在气泡外侧并与气泡平行，角色消息在气泡右边，我的消息在气泡左边。
- **一对一不显示每条气泡头像**，头像只在头栏。头像一律**圆形**。
- 头栏：头像 + 名字 + **状态行**（平时蓝色「在线」，生成时「正在输入…」）。
- 聊天背景：**Telegram 纯色底**（亮/暗各一）。
- **日期分割线**（今天/昨天/日期）。
- 输入栏：**表情按钮放在「+」和输入框之间常驻显示**；图片/转账/礼物/语音/历史收进「+」号，点开展开；主发送键 = **圆形纸飞机**；保留「输入」键（发消息但不触发 AI，用于铺垫连发）做成次要按钮。
- 表情面板：点击表情按钮打开；点击任意面板外区域关闭；emoji-picker 按需懒加载，避免打开 QQ 时卡顿。
- 会话列表：圆头像 52px、名字加粗、**末条消息时间（现在做）**；**未读红点（等"主动消息"功能再做）**。
- 字体：**全局保留用户的宋体，不覆盖**；字号只控制正文、不波及图标（已修）。
- 范围：**整个 QQ** 都改。

## 二、消息交互

- 手机：**长按气泡弹菜单**；桌面：悬停/右键。
- 菜单项：重新生成、编辑、收藏（已有）+ 收藏右侧 `>` 生成新版本 + **删除** + **回复引用**（让 AI 知道在回应哪句）+ **多版本切换**（swipe 重生成，保留多个版本左右翻，回复组最后一个气泡下方显示 `← 1/3 →`）。一次 AI 生成出来的若干短气泡视为一个**回复组**，重新生成、生成新版本、左右切换都作用于整个回复组，而不是单个气泡；回复引用和编辑仍作用于单条气泡。重新生成、`>` 和版本切换只显示在当前聊天末尾 AI 回复组上。
- 长按/右键：操作图标条在气泡上方浮出；点击任意外部区域后图标条消失，气泡排列恢复。
- 编辑消息：点击编辑后在当前气泡内切换为 textarea，不使用浏览器弹窗。
- 批量删除：输入栏 `+` 工具区提供垃圾桶入口，进入删除模式；可逐条选择，也可勾选“从选中楼层往下”后一键选中该消息及其后的所有消息。
- **表情回应（reactions）**：在气泡上贴 ❤️👍😂，**且 AI 也会给我的消息贴**。

## 三、人设 / 账号系统

- 「账号 = 一个 user 人设」，字段：**名字 / 头像 / 状态 / 个性签名 / 性别 / 生日 / 人设提示词 / 切换账号备注**。
- 切换账号**只改"我"的身份**，好友和聊天记录**共享**（不分隔数据）。
- 暂不做角色绑定专属人设；选择某个 user 人设后，QQ 当前聊天统一使用这个 user 身份。
- 已完成：App 内新建，管理入口在「我」页；当前 user 人设 id 保存到 `data/qq/settings.json`；一个人设一个 JSON，保存在 `data/userpersonas/`。
- 已完成：发给 AI 的 user 信息只包含名字、性别、生日、用户人设；状态、个性签名、备注只用于 UI。

## 四、提示词架构（核心 · 预设驱动）

- **导入酒馆（SillyTavern）预设 JSON 作为装配模板**。已使用 `apps/prompt-manager/Liminal_online.json` 作为默认源，首次导入到 `data/presets/st-presets/` 工作副本。
- 按预设的 `prompt_order` 装配：普通条目用自身文字；**占位标记（marker）在其位置注入对应内容** —— 角色卡设定、用户人设、世界书、聊天历史等已接入。可用表情清单、NSFW 等仍待接入。
- 现有“硬编码拼接”已变为回退路径；正常情况下 QQ 回复优先走当前酒馆预设。
- **NSFW**：角色卡 nsfw 字段 → 提示词中**专门的 NSFW 小节**（填了才出现），尺度由用户模型决定。
- **世界书（lorebook）**：已支持全局常驻选择 + 角色绑定，并可导入酒馆世界书 JSON；当前是选中世界书全量注入，关键词/条件命中、优先级和 token 控制仍待做。
- **表情包**：App 自动把"可用表情清单（名字即含义）"注入提示词；AI 输出 `[表情名]`；前端在拆消息层识别并渲染成表情图（匹配不到则当普通文字）。媒体 `[内容.jpg]` 格式预留（等生图）。

## 五、AI 行为 / 内容

- 多条短气泡（提示词 + 后端拆分双保险）；**无旁白**（删括号）。
- **AI 会发 emoji 和表情包**（见上）。
- **流式打字机：不要**，整条直接出。
- 长期记忆：**自动总结旧内容**（后期）。
- 自定义**正则替换规则**清理/改写输出（后期）。
- **主动消息**：角色会"主动找你"（后期，陪伴感核心）。

## 六、角色卡

- 来源：**导入酒馆 PNG 角色卡 + App 内手动新建，都要**。
- **开场白**：进新聊天自动显示角色 first_mes；多开场白可 swipe。
- AI 配置：主 API 地址/Key/模型仍是全局一套；温度、TopP、惩罚和最大回复长度由 QQ 当前酒馆预设的采样参数控制，不按角色单独设。

## 七、其它

- 图片 vision：**已做**（仅最近 N 条图片附真实图像，省 token）。
- **一键备份/导出**全部数据（角色/聊天/人设）：要。
- 真语音（TTS，语音消息/读出来）：后期。
- 工作方式：**我自己用本地预览截图核对**，重要节点请用户把关。
